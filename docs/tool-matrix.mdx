# 工具矩阵

本书绝大多数误区是**范式级**的——它们来自大模型 + 工具调用 + 长上下文 + 自动执行这套共同机制，**换哪个工具都会踩**，所以默认每条都适用于各家 AI 编码工具。这页只索引**两类例外**：同一根因在不同工具里**防法不同**（写在条目的「工具差异」小节里），以及某个坑是**某工具独有机制**带来的（单独成条 / 进案例库）。

:::note 覆盖深度（请如实看待）
**Claude Code 覆盖最深**，是全书的默认参照（几乎每条都适用，故矩阵里不单设它一列）。**Gemini CLI、Codex CLI、Cursor、GitHub Copilot** 是已铺的第二批，下表是它们各自的首批工具差异——**Codex / Gemini（开源、官方文档全）最实**；**Cursor / Copilot（闭源、多形态、版本快变）以安全披露 + 官方文档为主，深度更薄**。矩阵只写**机制名**、不写具体命令，且**很快过时**——真要动手，以你所用版本的官方文档为准。截至 2026-06。
:::

## 各工具一节（机制 + 独有条目）

### Gemini CLI
默认值最反直觉的一档：**Plan Mode 默认开、Checkpointing 默认关**。安全面有两起在野高危：Tracebit RCE、CI 自动信任工作区（GHSA-wpqr，CVSS 10）。
- 独有条目：[Gemini 信任继承](./00-setup-collaboration/gemini-folder-trust-inheritance.mdx)
- 案例：[一句「介绍下这个 repo」被劫持（Tracebit RCE）](./cases/gemini-cli-tracebit-rce.mdx)
- 全部条目 → [/tags/gemini-cli](/tags/gemini-cli)

### Codex CLI
唯一**默认即沙箱**（macOS Seatbelt / Linux Landlock + 网络关）的一档，与 Claude Code「沙箱默认关」正好相反；配置分 `AGENTS.md`（建议）/ `config.toml` / `requirements.toml`（org 强制）三层。
- 案例：[一个 `.codex` 配置把「打开仓库」变成静默 RCE（CVE-2025-61260）](./cases/codex-cli-config-rce.mdx)
- 全部条目 → [/tags/codex-cli](/tags/codex-cli)

### Cursor
门锁最软：官方明说 `permissions.json` 与 `.cursorignore` 都是「尽力而为、不是安全保证」；命令闸多次在**解析层**被绕（denylist 已废、NomShub）。MCP 出过双 CVE（CurXecute / MCPoison）。
- 独有条目：[.cursorignore 不是安全边界](./00-setup-collaboration/cursor-ignore-best-effort.mdx)
- 全部条目 → [/tags/cursor](/tags/cursor)

### GitHub Copilot
三态（补全 / Chat / coding agent）各自防法不同：coding agent 有**服务端护栏**（只推 `copilot/*` 分支、默认开防火墙），但有**点名缺口**（防火墙不覆盖 MCP 与 setup 步骤）；指令文件 code review 仅读前 4000 字。安全面有 CVE-2025-53773、CamoLeak（CVSS 9.6）。
- 案例：[私有库密钥经 Camo 图片代理逐字外带（CamoLeak）](./cases/github-copilot-camoleak.mdx)
- 全部条目 → [/tags/git-hub-copilot](/tags/git-hub-copilot)

### Claude Code
全书**默认参照**，几乎每条都适用，故不单设 tag / 不占一列。它的强制层最硬：回溯快照默认开、`permissions.deny` 客户端硬拦截、hooks 确定性执行——上面四个工具的「工具差异」多半是在和这套默认作对比。

:::note 这张表会过时
工具差异和默认值随版本变化很快。这里只记机制名与方向、不记具体命令；真要动手，以你所用版本的官方文档为准。发现已过时，欢迎在 [GitHub](https://github.com/xuebinma/AIWiki/issues) 指正。
:::
